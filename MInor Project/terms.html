Terms & Conditions

These demo terms describe how the CareFinder sample application should be used. When you adapt this project for a real hospital or institution, replace this page with an agreement reviewed by your legal team.

1. Non-production use

The CareFinder codebase is provided for educational and prototyping purposes. It is not intended to be used as-is for managing real patient care, emergency triage, or hospital operations.

2. No medical advice

The application does not provide medical advice, diagnosis, or treatment. All clinical decisions must be taken by qualified healthcare professionals in accordance with local protocols.

3. Data accuracy

Hospital lists, bed counts, ambulance ETAs, and tariffs are demo values only. When you connect this project to your own data sources, you are solely responsible for:

  • Ensuring that the information displayed is accurate and regularly updated.
  • Communicating clearly to users which values are live and which are approximate.
  • Auditing changes made by staff through the admin portal.

4. Emergency usage

In real emergencies, patients and families should always be advised to call the official local emergency number first. CareFinder, even when integrated with live ambulance systems, must be positioned as a supporting tool.

5. Security & compliance

The sample backend includes only basic authentication and file-based storage. Before using it in production you must:

  • Implement secure hosting, encryption, logging, and monitoring.
  • Follow your organisation's information security policies.
  • Comply with all applicable laws and health regulations.

6. Modifications

You are free to modify the source code to suit your project. However, any institution deploying a modified version should maintain its own change log, release notes, and support contacts for end users.
