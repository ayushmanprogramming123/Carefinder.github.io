Privacy Policy

This privacy policy describes how the CareFinder demo application handles information. When you adapt this project for production, you must review and customise this document with your institution's legal and compliance teams.

1. Data collected in the demo

The current demo does not persist clinical data such as diagnoses, prescriptions, or medical history. It only processes:

  • Location queries used to search nearby hospitals.
  • Demo ambulance requests (pickup text, contact number, optional hospital name).
  • Staff login details and bed counts in a local JSON file for testing.

2. How staff data is handled

In the sample Node.js backend, staff user accounts (email, password hash, linked hospital) are stored in a local file backend-data.json.

  • Passwords are hashed using bcrypt.
  • No real-world encryption at rest, backups, or key rotation are implemented. You must add them before production use.
  • JWT tokens are used for staff authentication during a browser session.

3. Location & geocoding

When you use features like "Use my location" or ambulance pickup maps, your browser may share approximate geographic coordinates. These are used only to:

  • Center the hospital or ambulance map near your position.
  • Call public geocoding services (OpenStreetMap Nominatim) for address lookup.

You should update this section if you proxy or log these requests through your own infrastructure.

4. Cookies & local storage

The demo frontend uses browser local storage for convenience—for example, to cache a selected hospital profile between pages. No tracking or analytics cookies are added by default.

If you integrate analytics, consent banners, or third-party SDKs, describe them clearly here and implement appropriate consent flows.

5. Using this template in production

Before using CareFinder in a real hospital environment, you should:

  • Complete a formal data protection impact assessment.
  • Define retention periods and deletion processes for staff and patient data.
  • Document data processors, hosting providers, and cross-border transfers.
  • Ensure compliance with Indian and local regulations for medical data.
